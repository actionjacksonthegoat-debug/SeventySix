Loading State Guidelines

Use Skeleton Loaders for data fetching (read operations) and Spinners for user actions (write operations like save, submit, delete).

view_day Skeleton Loaders For Load/Fetch Events
  • ✓ Initial page/component load
  • ✓ Data fetching (GET requests)
  • ✓ Lazy-loaded content
  • ✓ Image/media loading
  • ✓ Iframe content loading

Why: Skeletons preview content layout, reducing perceived wait time and preventing layout shifts.

sync Spinners For Save/Action Events
  • ✓ Form submissions
  • ✓ Save/Update operations
  • ✓ Delete confirmations
  • ✓ User-triggered actions
  • ✓ Background processing

Why: Spinners indicate ongoing user action without implying content structure, appropriate for write operations.
